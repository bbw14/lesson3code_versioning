Highlight #1 What is a Version Control System?

A Version Contol System (VCS) is a system that essentially backs up your files and only save the changes every time. The VCS looks for changes in your files and records each change but never overwrites the last file you had. This in turn keeps these different "versions" all separate so if you were to want to go back a few changes you can do so using a VCS. This also helps those who are working together on the same project. You aren't writing over someone else's work you are both working separately on the same files. When they are saved they are saved as different files of the same original not overwriten.

Highlight #2 Why use Version Control Systems?

Version Control Systems are used first off to back up your files you are working on. VCS are also used by people who are collaborating on files VCS save each different version separately therefore you can have as many people working on the same file and each person save it as its own work and then each person can view each piece of the project. You can also use VCS to restore a later version of what you are working on even if you saved it because version control systems save each version seperatly.

Highlight #3 Basic Git Commands

The basic Git commands are how you get Git to do what you want and work with Codio as well. In Codio to move files into your staging area (shopping cart) you use git add . this moves the files so they are then able to move them to your local repository. To move files from the staging area to the local repository you use the git command git commit -m 'tag' tag is replaced with what you want the version to be called. From the local repository you move files to your GitHub repository by using git push this pushes what you have in your local repository into your GitHub repository.
